# 恭喜完成本章实验！

你已经掌握了 Rego 中迭代的核心用法：

- **`some item in collection`** —— 迭代集合、数组、对象的值，实现"存在量化"语义
- **`some key, value in collection`** —— 键值对迭代，同时获取索引/键与值
- **`_` 匿名变量** —— 忽略不关心的迭代维度
- **嵌套迭代** —— 多个 `some ... in` 组合，检查多层嵌套结构
- **自由迭代** —— `some id, index` + 路径访问，或用 `_` 作为匿名迭代变量

记住 Rego 迭代的核心语义：只要集合中**至少有一条路径**能让所有条件同时成立，规则就成功——这是"存在量化"，而不是"全量检查"。

下一章我们将学习**集合声明与生成表达式（comprehension）**，用更简洁的语法从现有集合中过滤或变换出新的集合。
