# 🎉 实验完成！

你已经成功完成了规则条件详解与小测验的实验，掌握了以下核心概念：

| 条件类型 | 核心规则 |
|----------|---------|
| 常规条件 | 求值成功且不为 `false` 即成功；Rego 中没有 falsy（`0`、`""`、`null`、`[]` 都是成功的） |
| 逻辑非条件 | `not` 反转成功/失败；`not 0`、`not ""`、`not null` 都是**失败**的 |
| 赋值语句条件 | 赋值操作成功即条件成功；`var := false` 是成功的！ |
| 集合成员条件 | `in` 检查成员/值；`some key, _ in obj` 检查键存在；避免用 `array[i]` 检查存在性 |

## 关键要点

- **Rego 中只有 `false` 和 undefined 会让条件失败** —— `0`、`""`、`null`、`[]`、`{}` 都不是失败
- **赋值 `false` ≠ 条件 `false`** —— `var := false` 成功，但后续 `var` 作为条件会失败
- **`in` 检查的是值不是键** —— 对于 object，`"key" in obj` 会失败，应该用 `some "key", _ in obj`
