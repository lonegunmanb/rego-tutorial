# 恭喜完成本章实验！

你已经掌握了 Rego 中与类型和比较相关的常见陷阱：

- **集合/对象比较**：集合与集合、对象与对象之间比较大小的结果不可靠（成员顺序不确定），只能用 `==` 判等
- **跨类型比较**：OPA 中不同类型可以比较大小（`null < boolean < number < string < array < set < object`），`"0" > 3` 为 `true`。务必在比较前检查类型，或使用类型安全比较函数
- **部分定义默认值**：部分定义规则在所有条件都不成立时，会自动产生空集合/空对象，而不是未定义值。空集合作为条件会被视为 `true`
- **命名遮蔽**：局部变量会遮蔽全局变量，自定义函数会遮蔽内建函数和操作符。避免使用 `gt`、`lt`、`replace` 等内建名称

**关键要点**：编写生产级 Rego 策略前，务必留意这些类型与比较相关的陷阱，它们可能导致策略在某些边界情况下产生完全相反的结果。
