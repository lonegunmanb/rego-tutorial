你已经掌握了 Rego 中与引用和迭代相关的常见陷阱：

- **未声明迭代变量**：未经 `some` 声明的变量可能与包级变量冲突，导致迭代变成固定索引访问。始终用 `some` 显式声明迭代变量
- **无效路径引用**：访问不存在的字段会导致规则静默失败。使用 `exists(x) if { x == x }` 安全判断路径是否存在，使用 `not x == x` 判断路径不存在
- **错别字**：拼错的字段名不会引发语法错误，只会静默失败。务必编写单元测试覆盖各种分支
- **输入数据预校验**：在执行策略规则前先校验输入数据的结构和类型，避免脏数据引发意外行为
- **反转含迭代规则**：`some item in arr; item != "a"` 的含义是"存在一个不是 a 的元素"，而不是"不包含 a"。正确做法是用 `not` 反转正向规则，或用 `every`

**关键要点**：Rego 的很多陷阱源于其声明式语言的特性——无效引用不报错、迭代变量可能被意外绑定、逻辑反转需要特别小心。编写生产级策略时，务必牢记这些陷阱。
