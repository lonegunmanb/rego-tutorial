# 🎉 实验完成！

你已经成功完成了路径引用与 Package 的实验，掌握了以下核心概念：

| 概念 | 说明 |
|------|------|
| 无效路径引用 | 访问不存在的路径不会报错，而是返回 undefined（等同于 `false`） |
| 隐含条件 | 每个路径引用都隐含了"该路径必须存在"的前提条件 |
| `not input.x` | 可以判定路径不存在，但无法区分"不存在"和"值为 false" |
| `not input.x == input.x` | 更稳妥的判定路径是否存在的技巧 |
| `package` | 以包为单位组织代码，同名包跨文件自动合并 |
| `import data.` | 跨包引用规则，支持完整路径和简写两种方式 |

## 关键要点

- **无效路径引用是静默的** —— 不会报错，规则会直接不成立。编写安全策略时要特别小心空输入绕过检查的情况
- **`not x == x` 技巧** —— 用于准确判定路径是否存在，不受值为 `false` 的干扰
- **同名 package 自动合并** —— 不同文件中声明相同 `package` 的代码会被合并到同一命名空间
