# 恭喜完成本章实验！

你已经掌握了 Rego 中**部分定义（Partial Definition）**的核心技术：

- **`contains` 部分定义 Set**：用多条 `contains` 语句逐个向集合添加元素
- **`[]` 部分定义 Object**：用 `obj[key] := value` 逐条向对象添加键值对
- **v0 `deny[msg]` 的本质**：它是部分定义集合的旧语法，v1 中应使用 `deny contains msg if { ... }`
- **批量部分定义**：结合 `some` 迭代和条件，一条规则可以贡献多个成员——这是编写 conftest 策略的核心模式
- **复杂表达式**：部分定义中可以使用函数调用、嵌套结构等任意 Rego 表达式
- **数组限制**：Rego 不支持部分定义数组，需要构建数组时请使用 Array 生成表达式

**关键要点**：部分定义的集合/对象在没有任何规则成功时是**空集合/空对象**，而不是 `undefined`。这与完整定义规则的行为不同，我们将在后续"常见陷阱"章节中详细讨论这一差异。
